System Features:

3 user roles with proper permissions
2 restaurant branches (Keonbae_KB & Keonbae_RW, both in London)
120 items across 7 categories with SKUs
Full order workflow (place → approve → dispatch → receive)
Stock management with units (kg, litre, box, crate)
Low stock alerts with reason tracking
Stock movement audit log
Order rejection with reasons
Order archiving system
Dark mode toggle
Category filtering and SKU search

Infrastructure:

PostgreSQL database on Supabase
Express.js backend on Vercel (serverless)
HTML/JS frontend on Vercel
GitHub repository for version control
JWT authentication
CORS configured
Auto-deploys on git push
